# PiBoard Design Language

> **站牌为骨，复古科技为皮，统一语法，易扩展**
>
> 70% railway display language · 20% retro electronic product feel · 10% restrained futuristic accent

## 文档定位

本文档定义 PiBoard 的项目级视觉方向和设计判断原则，用于统一后续页面、内容 Provider、布局与渲染调整的审美基准。它不是对当前代码实现的逐条验收清单，也不是要求所有已有页面立即完全符合的整改列表。

后续实现应在不背离本文档方向的前提下，结合硬件性能、可读性、数据长度和现有架构做工程取舍；当设计原则与实现限制冲突时，应优先保留「英国铁路电显作为骨架」这一核心识别度，再逐步向本文档靠拢。

---

## 1. 产品定位

一台 7 寸竖屏桌面信息终端，以英国铁路电显风格为视觉骨架，将个人日程、天气、列车时刻等私人信息统一呈现在同一套点阵语言下。

---

## 2. 视觉路线

本项目不是铁路时刻表的复刻，而是**借用铁路电显的视觉文法表达个人信息**。  
铁路感提供骨架（网格、层级、节奏），复古科技感提供皮肤（发光、圆点、暗底），克制的未来感控制边界（不跑偏为赛博朋克，不堆砌装饰）。

三层比例是方向性约束，不是装饰比例参考：
- **70% railway display**：点阵字体、黑底、顶部 header 栏、行列式内容、翻页节奏、ticker 跑马灯
- **20% retro electronic**：琥珀橙主色、暗点背景、轻微发光晕、复古仪器感的状态标签
- **10% restrained futuristic**：单像素分割线、极简留白、状态颜色语义化（绿/橙/红）

---

## 3. 必须保留的 Railway Display 特征

以下是这套语言的基因，缺少任何一项都会失去辨识度：

| 特征 | 说明 |
|------|------|
| **点阵字体** | 所有文字使用 bitmap 圆点点阵渲染，不使用 TTF 字体 |
| **暗点背景** | 未点亮的像素必须渲染为 `#1C1000`，不是纯黑；这是 LED 感的核心 |
| **极深背景** | 整体背景 `#080808`，视觉上接近熄灭的 OLED 像素 |
| **顶部 header 栏** | 每块板左上角时间/日期、右上角位置/标题，模拟站牌顶栏 |
| **行列式内容** | 内容区用左列（标签/站名）+ 右列（值/时间）的两栏对齐布局 |
| **底部 footer + 状态** | 底部信息来源 + 右侧状态标签（On time / Delayed / OK），dim 色 |
| **ticker 跑马灯** | 底部从右至左滚动的补充信息，循环播放 |
| **翻页节奏** | 内容超出区域时上滑翻页，不截断，不缩小字号 |
| **全大写倾向** | header、footer、状态标签优先全大写，行内容按实际数据决定 |

---

## 4. 可以引入的 Retro-Tech 特征

这些特征在 20% 的权重内合理使用，增强「复古电子产品」质感：

- **发光晕**：亮点圆心额外叠加更小的高亮圆（`#FFD080`），模拟 LED 发光漫射
- **仪器感分割线**：单像素水平/垂直分割线，颜色用 `dim` 色而非亮色，不要装饰性
- **状态标签样式**：右对齐的状态文字（`ON TIME`、`DELAYED`、`2 UPCOMING`）配合颜色语义，模拟仪器指示灯
- **轻微数字翻转动画**：时间数字更新时有纵向翻牌感，模拟机械翻牌板（可配置关闭）
- **亮度衰减感**：整体亮度可调，低亮度下点阵圆点颜色自然衰减，保留暗点对比

---

## 5. 明确禁止的跑偏方向

以下方向会破坏设计一致性，不得引入：

| 禁止项 | 原因 |
|--------|------|
| **彩色渐变背景 / 毛玻璃效果** | 破坏黑底 LED 感，跑向现代 UI |
| **圆角卡片 / 阴影盒子** | 这是 Fluent/Material 语言，不属于此风格体系 |
| **图标（emoji / SVG icon）** | 点阵屏没有图标；用文字标签替代 |
| **多种字体混用** | 只有一套点阵字体，不引入 TTF 混排 |
| **动态粒子 / 扫描线特效** | 赛博朋克方向，超出 10% futuristic 配额 |
| **高饱和多色同屏** | 颜色仅用于语义（状态），不用于装饰；同屏不超过 3 种颜色 |
| **纯黑背景（无暗点）** | 失去 LED 点阵物理感 |
| **TTF 字体 fallback 替换点阵** | 视觉上立即跌出 railway 语言 |
| **天气 / 日程图标化表达** | 个人信息也应被翻译为站牌数据语言，而不是常规 dashboard 卡片 |
| **复杂动画 / 粒子爆炸** | 超出硬件性能预算，且破坏克制感 |

## 6. 页面体系

竖屏（600×1024）下的各页面定位与视觉重心：

### Overview（总览页）

- 作用：一屏呈现最重要的跨域摘要（下一个行动 + 今日状态 + 天气/交通/日程摘要）
- 布局倾向：单板全屏，中高密度，先给出唯一判断，再给出统一列表
- 视觉重心：title 区必须是唯一 Hero / Anchor，例如 `NEXT 09:30`、`TODAY 3 EVENTS`、`SERVICE ALERT`
- 时间/日期：作为 header 稳定锚点显示，不默认抢占 title 区
- 不允许：将多页面硬塞进一屏，不得把 Overview 做成 dashboard 卡片网格

### Rail（列车时刻页）

- 作用：标准报站板，展示下一班次目的地、中间停靠站、出发时间、状态
- 布局倾向：与真实 departure board 最接近，header 必须显示当前时间和站台号
- 视觉重心：`title` 是目的地大字，`rows` 是停靠站列表，`status` 是 `ON TIME / DELAYED`
- 特别规定：列车延误状态用 `red`，准点用 `green`，不得用其他颜色替代

### Weather（天气页）

- 作用：天气信息以「站牌数据卡」形式呈现，而非图标卡片
- 布局倾向：header 显示当前温度和城市，title 是天气描述文字，rows 是湿度/风速/能见度等
- 视觉重心：信息数据化而非可视化，用文字和数值，不用天气图标
- 特别规定：状态摘要反映天气描述，颜色倾向使用中性高亮（如 `white`），不把晴雨图标作为主表达

### Schedule（日程页）

- 作用：今日日程 + 未来几天预览，以时刻表节奏呈现
- 布局倾向：header 显示星期和日期，title 必须有信息价值，例如 `TODAY`、`3 EVENTS`、`NEXT 09:00`
- 视觉重心：当前最近的事件应有唯一高亮，往后的事件用 `dim` 色退场
- 特别规定：超出今天的条目必须用 `dim` 颜色标记，与今天内容形成明显对比

### Todo / Info（待办 / 自定义页）

- 作用：个人 todo、备忘录、公告等非结构化信息
- 布局倾向：与 Rail 页结构相同，但内容完全由用户自定义
- 视觉重心：保持与其他页一致的字号和行列格式，不允许为「富文本」感改变布局
- 特别规定：即使是自定义内容，也必须严格遵守本文档的颜色和字号规则

## 7. 颜色使用原则

### 调色板

| 名称 | HEX | RGB | 用途 |
|------|-----|-----|------|
| `amber` | `#FF9500` | (255, 149, 0) | **主色**：正文、标题、主要信息 |
| `dim` | `#1C1000`（暗点）/ `#503718`（文字） | — | 暗点背景 / 次要文字、footer、过去事件 |
| `green` | `#50DC50` | (80, 220, 80) | **正常状态**：On time、OK、active |
| `orange` | `#FFA01E` | (255, 160, 30) | **注意状态**：即将到期、次要警告 |
| `red` | `#DC3232` | (220, 50, 50) | **告警状态**：Delayed、错误、逾期 |
| `white` | `#DCDCDC` | (220, 220, 220) | **中性高亮**：强调、天气描述、特殊项 |
| `bg` | `#080808` | (8, 8, 8) | 全局背景 |
| `highlight` | `#FFD080` | — | 亮点圆心高光，仅用于点阵发光效果 |

### 使用规则

1. **主色 `amber` 承载 80% 以上的屏幕文字**，其他颜色仅用于语义区分
2. **状态颜色（green / orange / red）只用于状态摘要和关键数值**，不用于装饰性染色
3. **同屏同时出现的颜色种类 ≤ 3**（`amber` + 最多 2 种辅助色）
4. **`dim` 色文字用于已过去的、次要的、来源性的信息**，不用于主体内容
5. **避免亮色背景块**：背景以暗底和暗点为主，必要的状态/高亮底色必须低亮度、低面积、服务信息层级

## 8. 字体与字号原则

### 字体

主字体使用一套自定义 bitmap 点阵字体，两种规格：
- **5×7 点阵**：用于正文行、footer、header 辅助信息
- **7×9 点阵**：用于 title、重要数字、大号时间

英文、数字和常用符号走内置 bitmap 字模。中文等未覆盖的 Unicode 字符优先直接渲染项目内置的点点像素 Circle 字体（`third_party/fonts/dotted_songti/DottedSongtiCircleRegular.otf`，GPL-2.0），保留字体自身的小圆点结构，并将圆点点亮为实色 LED。fallback 字体仍可通过 `PIBOARD_CJK_FONT` 指向具体字体文件；若 Circle 不可用，则回退到 Square 版，再回退到 Pi 系统 CJK 字体（例如 `/usr/share/fonts/truetype/droid/DroidSansFallbackFull.ttf`）。

### 字号枚举与用途

| 字号级别 | 规格 | 典型用途 |
|----------|------|---------|
| `SMALL` | 5×7 | rows 内容行、footer、ticker |
| `MEDIUM` | 5×7（行距更松） | header_left、header_right |
| `LARGE` | 7×9 | subtitle、重要数据行 |
| `XLARGE` | 7×9（更大间距） | title、大号时间/温度 |

### 字号规则

1. **title 区永远用 XLARGE 或 LARGE**，不允许用 SMALL 节省空间
2. **rows 区永远用 SMALL**，不允许为了塞更多内容缩小字号
3. **同区域内字号统一**，不允许在 rows 内混用 SMALL 和 MEDIUM
4. **字号不做自由缩放**：过长内容优先用 ticker、翻页、截短或标题层级调整处理；title 可在预设大字号档位内选择，不压缩到正文级别

## 9. 信息层级原则

### 五区域层级（从上到下）

```
┌───────────────────────────────┐
│  HEADER  时间 / 位置           │  ← 第 5 层（最次要：当前时间与位置锚点）
├───────────────────────────────┤
│  TITLE   主信息                │  ← 第 1 层（最重要：目的地 / 天气 / 今日概览）
│  SUBTITLE 补充描述             │  ← 第 2 层
├───────────────────────────────┤
│  ROWS    结构化数据列表         │  ← 第 3 层（详细数据）
├───────────────────────────────┤
│  FOOTER  来源 · STATUS 状态    │  ← 第 4 层（元信息：数据来源和当前状态）
├───────────────────────────────┤
│  TICKER  补充滚动信息           │  ← 第 5 层（最次要：非关键补充）
└───────────────────────────────┘
```

### 层级规则

1. **用户视线第一落点是 TITLE**，TITLE 必须是最大字号且信息量适中（不超过一行）
2. **STATUS 是唯一带颜色语义的摘要**，一眼判断当前状态（正常 / 注意 / 告警）
3. **ROWS 是密度区**，行数由翻页机制管理，不压缩以放更多行
4. **HEADER 和 FOOTER 是锚点和元信息**，次于内容，颜色用 `amber`（header）和 `dim`（footer）
5. **TICKER 是可选的第六感知层**，放无关紧要的上下文信息；空内容时不渲染该区域

## 10. Do / Don't 清单

### Do ✓

- 用点阵圆点渲染所有文字，包括数字和标点
- 渲染所有暗点（`#1C1000`），即使字符为空的区域也要有暗点网格
- 保持 header 区的「时间 + 地点/标题」固定格式
- 状态颜色（green/orange/red）只用于表示数据的健康状况
- 超出的内容行用翻页处理，翻页动画使用上滑 ease-in-out
- ticker 文字从右向左，速度均匀，不做变速
- 分割线使用单像素，颜色用 `dim`，不加装饰
- 新增 Provider 时先把信息翻译成站牌式结构，再考虑具体字段和渲染实现
- 当前最重要的一行只允许出现一个高亮（下一班次、最近事件、最高优先级提醒）
- dim 色用于退场信息（历史、非今日事件、来源说明）

### Don't ✗

- 不使用 TTF 字体或任何非点阵渲染方式
- 不用彩色作为装饰（如给不同 Provider 配不同主色调）
- 不让背景出现任何非 `#080808` 的区域填充
- 不添加圆角、阴影、边框高光等现代 UI 元素
- 不在 rows 区混用不同字号
- 不为了塞更多内容而缩小字号或压缩行距
- 不同屏出现超过 3 种颜色
- 不在 title 区放多行文字，超出用 subtitle 承接
- 不让 ticker 消失在视野中后跳回（必须循环平滑）
- 不用 emoji 或 Unicode 图形符号替代文字标签
- 不堆砌动画效果；每次内容变化只有一个主动画
- 不在 Weather / Schedule 页引入图标式表达
